# Security Policy

## Supported Versions

We release patches for security issues for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it responsibly:

1. **Do not** open a public GitHub issue for security-sensitive bugs.
2. Open a **private security advisory** in this repository: go to **Security** → **Advisories** → **Report a vulnerability**.
3. Include a clear description, steps to reproduce, and impact if possible.
4. Allow time for a fix before any public disclosure.

We will acknowledge your report and work on a fix. We appreciate your effort to disclose issues responsibly.
